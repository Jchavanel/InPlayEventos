Para quién

Una propuesta más específica por tipo de usuario.

La web debe hablar distinto a jugador, organizador y patrocinador. Esa segmentación mejora conversión y percepción de valor.

Jugador

Quiere entender rápido qué se juega y cómo apuntarse.

Necesita fechas, sede, categorías, precio y una acción clara. El detalle de evento ya le acerca a la inscripción.

  • Información clara de torneo y categoría
  • Mejor percepción de calidad y organización
  • Base para horarios, cuadro y resultados
Organizador

Quiere una marca que proyecte orden y escalabilidad.

Necesita una presencia capaz de vender el evento hoy y sostener la operativa digital mañana.

  • Presentación sólida de la propuesta
  • Preparación para formularios y backoffice
  • Base válida para torneos y circuitos
Patrocinador

Quiere contexto, continuidad y visibilidad cuidada.

Necesita ver comunidad, eventos, sedes y una imagen coherente para valorar colaboración.

  • Entorno visual más premium
  • Espacio para sedes, premios y partners
  • Lectura comercial más clara del circuito

Base de producto

La web ya puede enseñar evento y orientar el siguiente paso.

El objetivo ya no es solo verse bien. Ahora la superficie pública empieza a orientar al usuario hacia una acción más concreta.

Siguiente decisión correcta

Convertir registros en gestión real desde la propia web.

Esta fase 2.1 añade una base de operación: acceso admin, consulta de preinscripciones y actualización de estados.

Fase 2.1

Panel básico para revisar preinscripciones y mover estados.

Esta capa ya no es escaparate. Permite acceso admin con email y contraseña, lectura de preinscripciones y cambio de estado de pending a reviewed o confirmed.

Acceso admin

Entrar con email y contraseña.

Usa un usuario existente de Supabase Auth cuyo email esté autorizado en las políticas RLS del panel.
